基本数据类型与引用数据类型

基本数据类型

6种：Number、String、Undefined、Null、Boolean、Symbol (ES 6 新增)

使用typeof操作符检测数据类型

是的没错！返回值中没有”null“

当使用typeof检测一个null类型时会返回object ，这是因为null其实表示的就是一个空对象指针，所以理所当然会返回object

引用数据类型

当然是Object 类型，还有它的几个实例，Array 类型、Date 类型、RegExp 类型 、Function 类型等。

引用数据类型的值是按引用访问的

当一个对象赋值给一个变量时，该变量所拥有的存储在堆中一个对象的指针。

作用域 （重点哟）

虽然在《高程》上关于作用域的篇幅仅仅两三页，但是对于作用域理解直接关系到理解闭包之类的概念，初次读还是有一定的理解难度，我看到3第三遍才渐渐有点眉目（→ →）

执行上下文（execution context）

执行上下文也叫执行环境，执行上下文定义了变量或者函数有权访问其他数据，每个执行上下文都有一个与之关联的变量对象（variable object 简称VO），环境中定义的所有变量和函数都保存在变量对象中。

所有函数的执行都是在执行上下文栈中完成的。

如图，首先全局执行上下文入栈，当执行流到changeColor()时，changeColor()被推入到执行上下文栈中，执行其中代码，到swapColors()这里，把swapColors()推入栈，继续执行swapColors(),没有新的执行上下文，执行完毕，执行上下文栈推出swapColors()，继续changeColor()，执行完毕执行上下文栈推出changeColor()。最后在窗口关闭时，推出全局执行上下文。

那么每个函数的执行上下文内部又是怎么执行的呢

上面是个很常见的面试题，考察的就是关于变量提升的知识点，但是究其原理可能就不清楚了。

当demo()进入执行上下文栈中，会创建一个demo()的执行上下文。

执行上下文的生命周期分为:

  • 创建阶段：创建变量对象、建立作用域、确定this指向
  • 执行阶段：函数引用、变量赋值、执行其余代码
